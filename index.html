O que é?

  • Um controle de versão de terceira geração
  • Uma ferramenta que protege você de você mesmo.
    Gerações anteriores:
  • Primeira geração: 1972
  • Segunda geração: 1990
  • Terceira geração: 2005
    Porém:
  • lentas
  • ocupavam espaço
  • centralizadas
    Curiosidades
  • Primeiro commit no primeiro dia de desenvolvimento no próprio git
  • Primeira versão em 7 de abril de 2005
    Mas qual a diferença entre o git e os outros Sistemas de controles?
  • Ele não vê os arquivos como arquivos, mas sim como um grande arquivo

Pra que serve?

  • Rastreamento do histórico
    • Controlar versão (WOW)
    • Reverter alterações
  • Descentralização
    • Desenvolvimento paralelo
    • Redundâncias
  • Segurança
    • Validar se seus arquivos são realmente seus
  • Performance
    • Não desperdiça espaço criando cópias
    • Compressão de dados

Configurando

Atributos básicos (obrigatórios): 
git config --{nível} user.name "Batman"

git config --{nível} user.email batman@hotmail.com
    Níveis:
  • system - todos repositórios, todos usuários
  • global - todos repositórios, usuário atual
  • local - repositório atual, usuário atual
  • file - arquivos específicos
Desfazendo: 
git config --{nível} --unset {propriedade}
Criando todas estruturas necessárias para usar git: 
git init
    Tarefa 1:
  • Criar um repositório
  • Iniciar git
  • Configurações básicas (a nível de repositório)

Commit


Visualmente
Serve para registrar alterações, podemos pensar em um commit como sendo o checkpoint do seu jogo favorito.
Mas antes, precisamos selecionar o que queremos salvar.
Staging Area
  • Serve para que saibamos quais arquivos foram alterados/criados/removidos.
  • Nos mostra qual o status atual de nossos arquivos em relação ao commit pai.
  • Com exceçao do primeiro commit que não possui pai

    • Mostra quais arquivos foram alterados 
    git status

    Mostra o que foi alterado nestes arquivos 
    git diff

    Add

      Pode ser feito de duas maneiras:
    • Manual
    • Iterativo
    Manual
    Por arquivo 
    git add {files}

    Todos arquivos - novos/alterações/exclusões 
    git add .
    Iterativo 
    git add -i

    Extremamente útil pra quando se tem muitos arquivos, mas nem todos deverão ser inclusos

    Commit 
    git commit -m "My awesome feature"

    ou 
    git commit

    Branches

      Como o prório nome diz, podemos trabalhar com um sistema de "galhos"
    • Fazer experimentos
    • Desenvolver multiplas coisas ao mesmo tempo
    • Tudo isso sem comprometer o código original

    Visualmente
    Criar uma nova branch 
    git branch {branch_name}
    Deletar branch 
    git branch -D {branch_name}
      Tarefa 3.1:
    • Criar uma nova branch e separar arquivos de estilo da tarefa 1

    Checkout

    O comando checkout serve (entre outras coisas) para navegação entre branches.
    Trocando de branch 
    git checkout {branch_name}

    Dica:


    Criar branch e fazer checkout para ela no mesmo comando: 
    git checkout -b {branch_name}

    Merge

    O comando merge é usado para juntar branches quando a tarefa de uma branch foi finalizada.
    E esse é um dos pontos fortes do git, ele "sabe" quais partes de um arquivo deve sustituir por outro, de maneira automática. 
    git merge {branch_name}

    Atenção


    No momento do merge, você deve estar na branches sem alterações e no comando de merge, invocar a branch que foi alterada.
      Tarefa 3.2
    • Fazer o merge da branch criada na branch master
    • Commit
    • Deletar Branch antiga

    Logs

    Serve para visualização de onde e quando aconteceram as coisas
      Usado com vários formatos:
    • oneline
    • diff
    • shortlog
    • graph
      De várias maneiras:
    • quantidade
    • data
    • autor
    • arquivo
    • intervalo de commits
    Comando principal 
    git log {file}

    Mostra hash, autor, data e mensagem
    Oneline 
    git log --oneline

    Mostra hash (abreviado) e mensagem 
    git log --oneline {branch1}..{branch2}
    Stat 
    git log --stat

    Mostra hash (abreviado), mensagem e quais arquivos foram alterados
    Graph 
    git log --graph

    git log --graph --oneline
      Outros parâmetros
    • -n {valor} - exibe o número de commits pasasdo
    • --author="{name}" - exibe todos os commits feitos pelo autor
    • --pretty=format: - faz praticamente qualquer coisa com os parâmetros corretos

    Blame


    git blame {file} -L {initial_line, final_line}

    Diff

    Indo um pouco além no comando diff podemos usa-lo para exibir mudanças entre commits 
    git diff {hash_inicial}..{hash_final}
      Tarefa 5.0
    • Exiba os arquivos alterados no último commit
    • Exiba o que foi alterado nos arquivos no último commit
    • Encontre as alterações feitas pelo seu colega no seu código através do git blame

    Desfazendo as coisas

    Revert
    A maneira segura de refazer as coisas, reverte as alteraçãos feitas em um commit, ou em um intervalo

    Visualmente
    Desfaz um commit 
    git revert {hash}

    Desfaz todos commits de um intervalo 
    git revert {hash_initial}..{hash_final}

    Do commit mais antigo para o mais recente
    Reset
    A maneira não tão segura de desfazer algo, volta para o momento antes do commit, deixando as alterações na área de staging.

    Visualmente 
    git reset {commit}
    Pode também ser feito de maneira agressiva, isso significa dizer que tudo que veio depois ainda não existe 
    git reset --hard {commit}
      Tarefa 6.0
    • Reverta as alterações feitas pelo seu colega
    • Exclua a sua reversão
