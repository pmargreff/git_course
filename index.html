Pra que serve?

  • Rastreamento do histórico
    • Controlar versão (WOW)
    • Reverter alterações
  • Descentralização
    • Desenvolvimento paralelo
  • Segurança
    • Validar se seus arquivos são realmente seus
  • Performance
    • Não desperdiça espaço criando cópias
    • Compressão super otimizada

    Mas qual a diferença entre o git e os outros Sistemas de controles?
  • Ele não vê os arquivos como arquivo, mas sim como um grande arquivo

Configurando

    Níveis:
  • system - todos usuários do sistema
  • global - usuário atual
  • local - válida apenas para o repositório atual
  • file - configurações e comandos a nível de arquivo
Atributos básicos (obrigatórios): 
git config --local user.name "Batman"

git config --local user.email batman@hotmail.com
Outros atributos: 
git config --local --unset user.name

Commit

Registrar uma (várias alterações), podemos pensar em um commit como sendo o checkpoint do seu jogo favorito. Mas antes, precisamos selecionar o que queremos salvar.

Status

  • Usado para verificar qual o status dos arquivos de acordo com os arquuivos originais
  • Serve para que saibamos quais arquivos foram alterados/criados/removidos.

Add

    Pode ser feito de duas maneiras:
  • Manual
  • Iterativo
Manual
Por arquivo 
git add file1.c file2.c

Todos arquivos - novos/alterações/exclusões 
git add .
Iterativo 
git add -i

Delete


git rm file1.c file2.c

  • Arquivos gerados de compilação
  • Arquivos de configuração

Caso deseje ignorar um arquivo que já esteja commitado você precisa excluir o arquivo do git

Cuidado para não confundir com: 
git rm file1.c

    Como o prório nome diz, podemos trabalhar com um sistema de "galhos"
  • Fazer experimentos
  • Desenvolver multiplas coisas ao mesmo tempo
  • Tudo isso sem comprometer o código original

Checkout

O comando checkout serve (entre outras coisas) para navegação entre branches.

Merge

O comando merge é usado para juntar branches quando a tarefa de uma branche foram finalizadas.
E esse é um dos pontos fortes do git, em um merge, ele "sabe" quais partes de um arquivo deve sustituir por outro, de maneira automática.
